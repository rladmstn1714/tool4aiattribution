# Requirement 버전이 여러 개일 때 시각화 제안

여러 버전(Earlier / Current)이 있는 requirement만 해당하는, 겹침을 줄이고 구분을 쉽게 하는 시각화 옵션들입니다.

---

## 액션 목록(Related Actions 패널) 처리

- **선택한 버전만 표시:** 패널은 **선택된 requirement 한 개**의 액션만 표시합니다. Earlier를 클릭하면 Earlier 액션만, Current를 클릭하면 Current 액션만 보입니다. (타임라인도 선택된 버전만 표시하도록 맞춤.)
- **버전 라벨:** 해당 requirement에 버전이 2개 이상일 때 패널 상단에 **"Viewing: Earlier"** 또는 **"Viewing: Current"** 를 표시해, 지금 보는 목록이 어느 버전 것인지 명확히 합니다.
- **다른 버전 보기:** Outcome 보드에서 다른 버전(Earlier/Current)을 클릭하면 그 버전의 액션으로 전환됩니다. 별도 탭/토글 없이 선택만으로 전환됩니다.

---

## 1. **버전별 서브레인 (Sub-lanes)**

**구상:** Assistant / User 레인을 각각 "Earlier" / "Current" 두 줄로 나눔.

```
Assistant  [ Earlier:  ○     ○        ]
           [ Current:    ○     ○   ○ ]
----------- Turn 1 --- 2 --- 3 --- 4 ---
           [ Earlier:  ○   ○          ]
User       [ Current:      ○     ○   ]
```

- **장점:** 같은 턴에 Earlier/Current가 같이 있어도 겹치지 않음. 버전별로 스캔하기 쉬움.
- **단점:** 레인 높이가 2배로 늘어남. 버전이 1개인 requirement는 한 줄만 쓰면 됨(조건부 레이아웃).

---

## 2. **같은 레인 안에서 세로 오프셋**

**구상:** 레인은 하나인데, Earlier 액션은 마커를 살짝 위로, Current는 아래로(또는 반대로) 배치.

```
Assistant     ○(E)     ○(C)   ○(E)     ← E=위, C=아래로 살짝
User             ○(C)     ○(E)   ○(C)
```

- **장점:** 레인 수는 그대로. 턴 기준으로 한눈에 보이면서도 E/C 구분 가능.
- **단점:** 같은 턴에 E/C가 둘 다 있으면 여전히 가깝게 붙어 있음(완전 분리보다는 덜 겹침).

---

## 3. **마커 스타일로만 구분 (겹침 허용)**

**구상:** 위치는 그대로 두고, Earlier = 테두리만/연한 색, Current = 채움/강한 색. 또는 Earlier = 작은 원, Current = 큰 원.

- **장점:** 레이아웃 변경 없음. 구현 단순.
- **단점:** 같은 턴·같은 레인에 E/C가 같이 있으면 여전히 겹쳐 보일 수 있음.

---

## 4. **“지금 보는 버전” 하나만 표시 (선택 초점)**

**구상:** 버전이 2개 이상인 requirement일 때만, 타임라인에는 **선택된 버전의 액션만** 표시.  
(선택 없으면 Current만, Earlier 선택 시 Earlier만.)

- **장점:** 겹침 없음. “지금 보고 있는 버전”에만 집중 가능.
- **단점:** 한 화면에서 두 버전을 동시에 비교는 어려움.

---

## 5. **버전이 있을 때만 2단 미니 타임라인**

**구상:** requirement에 버전이 2개 이상일 때만, 그 requirement용으로 **위: Earlier, 아래: Current** 두 개의 짧은 타임라인을 세로로 배치.

```
[ Requirement: "Add cafe..." (2 versions) ]
  Earlier  ○ ---- ○ ---- ○
  Current  ○ ---- ○ ---- ○ ---- ○
```

- **장점:** Earlier/Current가 완전히 분리되어 비교하기 좋음.
- **단점:** 전체 턴 타임라인과는 별도 UI가 필요함(requirement 카드/패널 안에만 쓰는 구성).

---

## 추천 조합

- **타임라인 하나로 통일하고 싶다면:**  
  **4번(선택된 버전만 표시)** + 필요 시 툴팁/범례에 “Viewing: Current” 같은 문구.
- **한 화면에서 두 버전을 같이 보고 싶다면:**  
  **1번(서브레인)** 또는 **2번(세로 오프셋)** 이 겹침을 줄이기에 유리합니다.

원하시는 방향(한 레인 유지 vs 레인 분리, 비교 필요 vs 한 버전만 보기) 알려주시면 그에 맞춰 구체적인 컴포넌트/데이터 구조까지 제안하겠습니다.
